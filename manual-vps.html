🚀 Deploy — Actualizar la app

Deploy completo (pull + build + restart)
cd /var/www/smmpanel
git pull origin main
docker compose up -d --build
✅ Usá esto cada vez que hagas cambios en el código y los pushees a GitHub.
Solo reiniciar contenedores (sin rebuild)
docker compose up -d
Forzar recreación de contenedores
docker compose up -d --force-recreate
✅ Usá esto cuando cambiaste variables de entorno en docker-compose.yml.
Verificar que el deploy fue exitoso
docker ps

🗄️ Base de Datos — PostgreSQL

Acceder a la consola de PostgreSQL
docker exec -it boostins_db psql -U boostins -d boostins
Ejecutar un archivo SQL (migraciones)
docker exec -i boostins_db psql -U boostins -d boostins < /var/www/smmpanel/database/migrations/004_tickets.sql
Ejecutar una query rápida sin entrar a la consola
docker exec -it boostins_db psql -U boostins -d boostins -c "SELECT * FROM users WHERE role = 'user';"
Backup de la base de datos
docker exec boostins_db pg_dump -U boostins boostins > /var/www/backup_$(date +%Y%m%d).sql
✅ Hacé esto antes de cualquier migración importante.

📋 Logs — Ver errores

Ver logs del backend (últimas 50 líneas)
docker logs boostins_backend --tail 50
Seguir logs en tiempo real (stream)
docker logs boostins_backend -f
✅ Presioná Ctrl+C para salir del stream.
Ver logs del frontend
docker logs boostins_frontend --tail 30
Ver logs de nginx (errores de conexión)
docker logs boostins_nginx --tail 30
Ver variables de entorno cargadas en el backend
docker exec boostins_backend printenv | grep -E "REFERRAL|SMTP|MP_"

👥 Usuarios y Balance

Ver todos los clientes con su balance
docker exec -it boostins_db psql -U boostins -d boostins -c \
"SELECT email, name, balance, created_at FROM users WHERE role = 'user' ORDER BY created_at DESC;"
Agregar saldo a un usuario
docker exec -it boostins_db psql -U boostins -d boostins -c \
"UPDATE users SET balance = balance + 1000 WHERE email = 'cliente@gmail.com';"
Ver historial de pedidos de un usuario
docker exec -it boostins_db psql -U boostins -d boostins -c \
"SELECT o.quantity, o.price, s.name, o.status, o.created_at
 FROM orders o
 JOIN services s ON o.service_id = s.id
 WHERE o.user_id = (SELECT id FROM users WHERE email = 'cliente@gmail.com')
 ORDER BY o.created_at DESC;"

⚙️ Servicios

Ver todos los servicios activos e inactivos
docker exec -it boostins_db psql -U boostins -d boostins -c \
"SELECT name, provider_service_id, price_per_unit, is_active FROM services ORDER BY platform, sort_order;"
Cambiar precio de un servicio
docker exec -it boostins_db psql -U boostins -d boostins -c \
"UPDATE services SET price_per_unit = 4.50 WHERE name = 'Instagram Followers – Premium 365d';"
Activar / desactivar un servicio
-- Desactivar
docker exec -it boostins_db psql -U boostins -d boostins -c \
"UPDATE services SET is_active = false WHERE name = 'Instagram Followers – Elite 365d';"

-- Activar
docker exec -it boostins_db psql -U boostins -d boostins -c \
"UPDATE services SET is_active = true WHERE name = 'Instagram Followers – Premium 365d';"

📦 Órdenes

Ver órdenes recientes de todos los usuarios
docker exec -it boostins_db psql -U boostins -d boostins -c \
"SELECT u.email, s.name, o.quantity, o.price, o.status, o.created_at
 FROM orders o
 JOIN users u ON o.user_id = u.id
 JOIN services s ON o.service_id = s.id
 ORDER BY o.created_at DESC LIMIT 20;"
Ver órdenes en progreso (pendientes de completar)
docker exec -it boostins_db psql -U boostins -d boostins -c \
"SELECT u.email, s.name, o.quantity, o.provider_order_id, o.created_at
 FROM orders o
 JOIN users u ON o.user_id = u.id
 JOIN services s ON o.service_id = s.id
 WHERE o.status IN ('pending','processing','in_progress')
 ORDER BY o.created_at;"
Cambiar estado de una orden manualmente
docker exec -it boostins_db psql -U boostins -d boostins -c \
"UPDATE orders SET status = 'completed' WHERE id = 'UUID-DE-LA-ORDEN';"
Reembolsar orden (devuelve saldo al usuario)
-- 1. Obtener precio de la orden
docker exec -it boostins_db psql -U boostins -d boostins -c \
"SELECT id, user_id, price, status FROM orders WHERE id = 'UUID-DE-LA-ORDEN';"

-- 2. Devolver saldo y marcar como refunded
docker exec -it boostins_db psql -U boostins -d boostins -c \
"UPDATE users SET balance = balance + PRECIO WHERE id = 'UUID-DEL-USUARIO';
 UPDATE orders SET status = 'refunded' WHERE id = 'UUID-DE-LA-ORDEN';"
✅ También podés hacer esto desde el panel admin en /admin → Orders → Refund.

🚨 Emergencias

La app no carga / nginx caído
docker compose down
docker compose up -d
El backend crasheó en loop
docker logs boostins_backend --tail 100   # Ver el error
docker restart boostins_backend
Limpiar imágenes y cachés Docker acumuladas
docker system prune -f
⚠️ Esto elimina imágenes no usadas. Los datos de la DB no se tocan.
Ver espacio en disco del VPS
df -h
Ver uso de RAM y CPU del VPS
htop
Rollback: volver a versión anterior del código
git log --oneline -10           # Ver commits recientes
git revert HEAD                 # Revertir último commit
git push origin main            # Pushear el rollback
docker compose up -d --build    # Deployar
🚨 Solo usá git revert si algo salió muy mal. Avisá antes de hacer esto.
